
# 🗂 部品管理システム：要件整理

## 🔵 管理対象データ
- 部品基本情報（部品番号、名称、型番、種類）
- 管理情報（プロジェクト、責任者、使用者、設置場所）
- 状態履歴（使用履歴、貸出履歴、廃棄情報）
- 購入情報（発注日、納品日、販売元、価格）

## 🔶 機能要件
1. 新規購入部品の登録  
2. 使用者・責任者変更  
3. 多条件検索（使用者、期間、プロジェクト、種類、設置場所）  
4. 貸出・返却  
5. 廃棄処理（status更新）
